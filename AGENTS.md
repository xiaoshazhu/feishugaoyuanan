# UI 设计指南

> **设计类型**: App 设计（应用架构设计）
> **确认检查**: 本指南适用于可交互的应用/网站/工具。

> ℹ️ Section 1 为设计意图与决策上下文。Code agent 实现时以 Section 2 及之后的具体参数为准。

## 1. Design Archetype (设计原型)

### 1.1 内容理解

- **目标用户**: 内部开发/验证人员，确认全栈框架运行状态与渲染能力
- **核心目的**: 告知系统就绪状态，建立「干净、可靠、待填充」的初始信任感
- **情绪基调**: 静谧专注 / 避免杂乱与过度装饰

### 1.2 设计方向

- **Design Style**: Muji 极简 — 纯白背景+极细边框+充裕留白，传达「便签纸」般的初始纯净感
- **Application Type**: Tool（单页状态验证工具）— 无导航、无侧边栏、垂直居中卡片
- **Aesthetic Direction**: 高对比度低饱和排版，动态时间作为唯一视觉脉搏

## 2. Color System (色彩系统)

> 基于「干净便签纸」语义推导，纯白底+深墨文字+绿色就绪信号，零装饰色。

**色彩关系**: 纯白底 + 深墨灰文字 + 低饱和绿状态色
**配色设计理由**: 验证页需零干扰，白色传递「待填充」，深灰确保可读，绿色仅用于就绪信号
**主色推导**: primary 取深墨灰而非品牌色，因本页核心任务是「确认运行」而非品牌展示
**使用比例**: 90% 白/浅灰中性 / 8% 深墨文字 / 2% 绿色状态点缀

### 2.1 主题颜色

| Token                | HSL 值            | 说明                           |
| -------------------- | ----------------- | ------------------------------ |
| `background`         | hsl(0, 0%, 100%)  | 纯白页面底色                   |
| `card`               | hsl(0, 0%, 100%)  | 卡片背景，与页面同色靠阴影区分 |
| `foreground`         | hsl(220, 10%, 15%)| 主标题与时间文字               |
| `muted-foreground`   | hsl(220, 5%, 45%) | 副标题与次要说明               |
| `primary`            | hsl(220, 10%, 15%)| 主文字色兼作视觉锚点           |
| `primary-foreground` | hsl(0, 0%, 100%)  | 主色上文字（本页未使用按钮）   |
| `accent`             | hsl(220, 5%, 96%) | hover/focus 反馈背景           |
| `accent-foreground`  | hsl(220, 10%, 15%)| accent 上文字                  |
| `border`             | hsl(220, 5%, 92%) | 极细分隔线                     |

### 2.2 导航区配色

- **基调关系**: 无全局导航，不适用
- **关键状态**: 不适用
- **边界与背景**: 不适用

### 2.3 语义颜色

| 用途       | HSL 值             | 衍生说明                     |
| ---------- | ------------------ | ---------------------------- |
| 成功/就绪  | hsl(145, 60%, 40%) | 状态指示器绿色对勾与文字     |
| 成功背景   | hsl(145, 50%, 96%) | 可选：状态行浅绿底色         |

## 3. Typography (字体排版)

- **Heading**: Inter + system-ui, -apple-system, sans-serif
- **Body**: Inter + system-ui, -apple-system, sans-serif
- **字体策略**: 几何无衬线确保数字等宽对齐；回退栈覆盖 macOS/Windows/Linux

## 4. Layout Strategy (布局策略)

- **导航意图**: 无全局导航；单屏垂直居中，禁止任何顶部/侧边栏
- **页面架构**: Flex 垂直水平居中 + 单卡片 `max-w-md`；整体 `min-h-screen`
- **响应式**: 移动端卡片宽度自适应 `w-[90vw]`，桌面端固定 `max-w-md`

## 5. Visual Language (视觉语言)

- **形态参数**: 圆角 `rounded-sm (0.125rem)` · 阴影 `shadow-sm` · 间距基调 `spacious`
- **识别签名**: 时间戳等宽数字 + 极细卡片边框 + 状态行绿色对勾前缀
- **装饰策略**: 零装饰元素，仅靠排版层级与阴影建立层次
- **动效原则**: 时间每秒刷新无过渡；hover 反馈 150ms ease-out
- **可及性**: 正文对比度 ≥ 7:1；状态绿在白底上 ≥ 4.5:1

## 6. Component Principles (组件原则)

- **状态完整性**: 卡片仅 Default 态；状态指示器仅就绪/加载中两态
- **层级清晰**: 标题 `text-3xl font-bold` / 副标题 `text-base text-muted-foreground` / 时间 `text-xl tabular-nums`
- **一致性**: 所有颜色引用 Color System token；禁止硬编码色值

## 7. Image Direction (图片与视觉资产，按需)

- **Image Role**: 无强制图片需求
- **Image Art Direction**: 优先通过排版、阴影和动态时间建立视觉记忆点
- **Image Prompt Keywords**: 无
- **Image Avoidance**: 禁止插图、图标装饰、渐变背景、品牌 Logo

## 8. 应避免 (Anti-patterns)

- ❌ 添加导航栏、Footer、侧边栏或任何非概要设计的结构元素
- ❌ 使用彩色渐变、插画、Emoji 替代绿色对勾状态指示器
- ❌ 时间戳使用比例字体导致数字跳动；必须 `tabular-nums`
